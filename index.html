Areas de GIT

Working Directory

El area de trabajo corresponde a la carpeta donde creamos, eliminamos y realizamos las modificaciones de los archivos de nuestro proyecto. Casi nunca utilizaremos la consola en dicha area, a excepcion de la creacion del Repositorio al utilizar el comando git init o querramos clonar un repositorio remoto.

Staging Area (INDEX)

El area de Control de Cambios corresponde a un archivo dentro de la carpeta .git que identifica a que archivos y carpetas les estamos haciendo seguimiento. Se utiliza para realizar las capturas de codigo (SNAPSHOT) que luego deberan ser confirmadas mediante un commit y de esta manera corroborar el estado de los mismos.

Repository (Local/Remote)

El almacen de confirmaciones se corresponde a una estructura de subcarpetas dentro de la carpeta .git que contiene las confirmaciones realizadas en forma de archivos BLOB (Binary Large Object) y se utilizan como referencia para deshacer cambios en el directorio de trabajo, asi como tambien para comparar y resolver conflictos.
